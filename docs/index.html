Getting Started

Quick Start

Three ways to use Ally Widget — pick whichever fits your stack.

HTML 
<!-- Configure before the script -->
<script>
  window.AllyWidgetOptions = {
    position:      'bottom-right',
    lang:          'en',
    primaryColor:  '#6366f1',
    poweredByText: 'My Company'
  };
</script>
<script src="dist/ally-widget.min.js"></script>
⌨️ Press Alt + A anywhere on the page to toggle the widget. Set keyboardShortcut: false to disable.
JavaScript 
import AllyWidget from 'ally-widget';

const widget = new AllyWidget({
  position:      'bottom-right',
  primaryColor:  '#6366f1',
  poweredByText: 'My Company',
  onFeatureToggle(key, enabled) {
    analytics.track('a11y', { key, enabled });
  }
});

widget.startAccessibleWebWidget();
HTML 
<script src="https://cdn.jsdelivr.net/npm/ally-widget/dist/ally-widget.min.js"></script>
💡 The CDN build auto-initialises. Configure via window.AllyWidgetOptions before the script tag, or via data-ally-* attributes on the script element.

Label & Icon Overrides

Rename any button or swap its icon — per feature, without touching the source.

JavaScript 
window.AllyWidgetOptions = {
  featureOverrides: {
    'bold-text': { label: 'Bold' },
    'text-to-speech': {
      label: 'Read Aloud',

    }
  },
  // swap the toggle button icon globally
  icons: {
    accessibility: '<svg>…</svg>'
  }
};
Features

Language

The widget ships with two languages. Users can switch at runtime using the language picker in the panel footer — their choice is saved to localStorage.

CodeLanguageNative name
enEnglishEnglish
neNepaliनेपाली
JavaScript 
window.AllyWidgetOptions = { lang: 'ne' };
🌐 If lang is not set, the widget detects the browser's preferred language and falls back to English if unsupported.
Features

Text to Speech

Uses the browser's native Web Speech API — no API key, no external service. When enabled, click any text element on the page to hear it read aloud.

⚠️ Nepali TTS requires Chrome or Edge with the ne-NP voice installed. Firefox and Safari have limited Nepali support.
JavaScript 
window.AllyWidgetOptions = {
  lang:               'ne',
  ttsNativeVoiceLang: 'ne-NP',          // BCP-47 voice tag
  ttsNativeVoiceName: 'Google नेपाली',  // optional exact name
  ttsRate:            0.9,              // 0.5–2
  ttsPitch:           1.0               // 0–2
};

If the requested voice is unavailable the widget falls back to the browser's default voice automatically.

Features

Event Callbacks

Hook into widget lifecycle events for analytics, state sync, or custom behaviour.

JavaScript 
window.AllyWidgetOptions = {
  onOpen() {
    console.log('Panel opened');
  },

  onClose() {
    console.log('Panel closed');
  },

  onFeatureToggle(key, enabled) {
    // key     — e.g. 'bold-text'
    // enabled — true | false
    analytics.track('a11y_feature', { key, enabled });
  },

  onReset() {
    console.log('Settings cleared');
  }
};

Features

Dev Mode

Append ?ally-dev=true to any URL to unlock the Annotations and Accessibility Report tools. These run an axe-core scan in the background and surface issues — hidden in production, available during development.

URL 
https://yoursite.com/page?ally-dev=true
✅ The axe-core library is loaded on demand only in dev mode. It adds zero weight to your production bundle.

function enqueue_ally_widget() {

    'ally-widget',

    'before'

}

Customization

Customization Guide

Every aspect of the widget is configurable without touching source code.

Colour theme

Pass any CSS colour value. The entire panel re-derives from these tokens — change one, the whole thing updates.

JS — dark brand example
AllyWidget.init({
  primaryColor:       '#e11d48',
  primaryColorLight:  '#fb7185',
  primaryColorDark:   '#9f1239',
  // dark panel
  backgroundColor:    '#0f172a',
  textColor:          '#e2e8f0',
  cardBackground:     '#1e293b',
  borderColor:        '#334155',
  // shape
  borderRadius:       '8px',
  buttonBorderRadius: '12px'   // 50% = circle
})

Customization

Full Example

Everything configured in one place.

JS
window.AllyWidgetOptions = {
  // placement
  position:         'bottom-right',
  offset:           [20, 20],
  size:             '56px',
  keyboardShortcut: true,          // Alt+A

  // language
  lang:             'ne',          // 'en' | 'ne'
  storageKey:       'myapp-ally',

  // branding
  poweredByText:    'My Company',
  poweredByUrl:     'https://mycompany.com',

  // theme
  primaryColor:     '#6366f1',
  borderRadius:     '12px',
  buttonBorderRadius: '50%',

  // features
  disableFeatures:  ['annotations', 'accessibility-report'],
  featureOverrides: {
    'text-to-speech': { label: 'Read Aloud' }
  },

  // TTS
  ttsNativeVoiceLang: 'ne-NP',
  ttsRate:           0.9,

  // callbacks
  onOpen()  { console.log('opened') },
  onClose() { console.log('closed') },
  onFeatureToggle(key, enabled) { console.log(key, enabled) },
  onReset() { console.log('reset') }
};
Publishing

npm Publish

Build all output formats, then publish to npm. The .npmrc reads your token from the environment automatically.

Shell 
# Set your token (from .env or CI secret)
export NPM_TOKEN=your_token_here

# Build all formats, then publish
npm run build
npm publish
🔐 Store your token in .env (gitignored). The project .npmrc reads ${NPM_TOKEN} automatically — your token never touches source control.
